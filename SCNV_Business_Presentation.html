The Business Challenge

Why We Built SCNV

⚠️

Unproductive Transfers

A significant portion of inter-plant and DC-to-DC stock transfers are lateral or reverse movements that add cost but deliver no value to the end customer. Manual identification is time-consuming and error-prone.

🔍

Lack of Real-Time Visibility

Supply chain teams lack a single dashboard that shows the health of the distribution network — from allocation efficiency to customer fulfilment accuracy — leading to delayed and reactive decision-making.

🤖

Manual Decision Bottlenecks

Decisions on whether to approve, re-route, or reject a transfer order require consulting multiple SAP tables, master data files, and people. This creates delays and inconsistent outcomes.

💡
Core Insight: We need an intelligent system that can automatically triage every stock movement, identify waste, suggest optimal routes, and present actionable insights — without requiring manual SAP lookups.

STO Classification Engine

How We Identify Waste in the Network

The SCM Analyst Agent applies 4 business rules to every incoming Stock Transfer Order. Each rule checks the type of movement and determines whether it adds value or creates waste.

R1 Plant → DC Checks if it follows a strategic lane. If yes → Productive. If not → Sales Over Forecast.
R2 DC → DC Lateral movement between distribution centres — always Unproductive. Root cause: Incorrect Deployments.
R3 DC → Plant Reverse movement — goods flowing upstream. Always Unproductive. Root cause: Incorrect Deployments.
R4 Plant → Plant Inter-plant. Classification depends on sourcing model (Single vs Dual) and direction.

Data Foundation

The Data That Powers SCNV

SCNV relies on four key master data sets — the same data sources your SAP system holds today. These are normalised and loaded into a cloud database for real-time querying.

  • 🏭
    Plant Master All manufacturing plants with country codes, regions, and sourcing capabilities — defining where products are made.
  • 📦
    DC Master (Distribution Centres) All warehouses and DCs with their capacity, geographic location, and downstream customer assignments.
  • 🏷️
    SKU Master Product catalogue with sourcing model (Single/Dual), shelf life, sourcing plants, and demand categorisation.
  • 🛤️
    Strategic Matrix Defines which Plant→DC lanes are "strategic" (approved corridors) and their current capacity utilisation.

Governance & Control

Human-in-the-Loop Decision Making

SCNV doesn't replace human judgement — it enhances it. Critical decisions are surfaced as alerts that require human approval before execution, ensuring full control over supply chain operations.

🔔 STO Re-routing Alerts

When an STO is classified as having a high-risk pattern (e.g., lead time violation, reverse flow), it is flagged for human review. Managers can Approve, Reject, or Override with a reason.

🔔 SO Re-routing Decisions

When a Sales Order is allocated to a sub-optimal plant, the system suggests re-routing to a better source plant. The supply chain manager reviews the recommendation and decides.

Advanced Capabilities

Graph Memory & Process Mining

🕸️

Graph Database Memory (Neo4j)

Maps the entire supply chain as a graph of interconnected nodes — Plants, DCs, SKUs, and the routes between them — enabling the system to:

  • ✓ Find alternative sourcing DCs when stock is available elsewhere
  • ✓ Understand upstream/downstream impact of routing changes
  • ✓ Provide contextual data to AI agents for smarter decisions
  • ✓ Preserve network topology knowledge across sessions
⛏️

Process Mining (Celonis EMS)

Connects to Celonis to validate AI decisions against real process execution data — ensuring compliant and conformant operations:

  • ✓ Conformance scoring — how well does actual execution match designed flows?
  • ✓ Lane efficiency — identifying consistently underperforming routes
  • ✓ Variant pattern analysis — detecting deviation from standard processes
  • ✓ Toggle on/off from the dashboard (Celonis EMS toggle)
Security & Access Control

Enterprise-Grade Security

🔐

Supabase Authentication

All users authenticate through Supabase (industry-standard JWT tokens). Every API request is verified before data is served — no anonymous access.

👥

Role-Based Access Control

Admin users can upload data and manage configurations. Standard users interact with the chatbot and view dashboards. Roles are enforced at both frontend and backend.

🛡️

Row-Level Security (RLS)

Database-level policies ensure users can only access data they are authorised to see. Even if a query is crafted manually, RLS prevents unauthorised data exposure.

Document Upload & Knowledge Ingestion

Authorised admins can upload supply chain documents (Excel, Word, CSV, JSON) which are parsed and embedded into the system's memory. This enriches the AI's context without manual data entry. Uploaded files are previewed in-app before ingestion.
